# FundHuntr — Mobile Guide

Build, run, and ship the **iOS and Android apps** for FundHuntr.

> Looking for the browser build? See [`WEB.md`](WEB.md).

---

## How the mobile app works

FundHuntr's mobile app is the **same code** as the web app, wrapped in a native iOS/Android shell by [Capacitor](https://capacitorjs.com/). One codebase, three targets: web, PWA, native app.

```
┌──────────────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────────────┐ │
│  │  WebView (iOS WKWebView / Android System WebView)│ │
│  │                                                  │ │
│  │     [ FundHuntr web app — HTML/CSS/JS ]          │ │
│  │                                                  │ │
│  └─────────────────────────────────────────────────┘ │
│                                                      │
│  Native shell:   Capacitor                           │
│  Native plugins: StatusBar, SplashScreen, Haptics,   │
│                  App (hardware back button), etc.    │
└──────────────────────────────────────────────────────┘
```

The web app talks to the native layer through `mobile.js`. In a browser it's a no-op. Inside Capacitor it:

- sets the status-bar style to green
- dismisses the splash screen after load
- fires a light haptic when you tap a swipe action / button / profile card
- wires Android's hardware back button to `history.back()`

## One-time setup

From the project root:

```bash
# Install Capacitor + plugins
npm install

# Add the native platforms — creates ios/ and android/ folders
npx cap add ios
npx cap add android

# Copy web assets into the native projects
npx cap sync
```

The `ios/` and `android/` folders are **gitignored** by default — they're regenerated by `npx cap add`. If you want to commit them (recommended for shipping to stores), remove the relevant lines from `.gitignore`.

## Run on a simulator / device

### iOS

```bash
npx cap open ios
```

This opens `ios/App/App.xcworkspace` in Xcode. From there:

1. In the top bar, pick a simulator (e.g. iPhone 15 Pro) or your tethered iPhone
2. Hit ▶︎ to build & run

Hot-iteration loop:

```bash
# Edit any HTML/CSS/JS in the project root
npx cap sync          # copies the new web assets into ios/App/App/public
# Hit ▶︎ in Xcode again — instant rebuild
```

### Android

```bash
npx cap open android
```

This opens the project in Android Studio. From there:

1. Wait for Gradle sync to finish (one-time, ~30s)
2. Pick an emulator (or your tethered device with USB debugging on)
3. Hit ▶︎ to build & run

Hot-iteration loop is identical: `npx cap sync` then re-run.

## Configuration

App identity and native plugins are configured in `capacitor.config.json`:

```json
{
  "appId": "com.fundhuntr.app",
  "appName": "FundHuntr",
  "webDir": ".",
  "plugins": {
    "SplashScreen": {
      "backgroundColor": "#15a36b",
      "launchShowDuration": 1500
    },
    "StatusBar": {
      "style": "DEFAULT",
      "backgroundColor": "#15a36b"
    }
  }
}
```

If you change `appId`, also update it in:

- **iOS**: `ios/App/App.xcodeproj` → General → Bundle Identifier
- **Android**: `android/app/build.gradle` → `applicationId`

(or just delete the platform folder and re-run `npx cap add ios|android` — Capacitor will read the new ID from the config.)

## App icons & splash

Capacitor uses your existing `manifest.json` icons as a starting point, but for the App Store / Play Store you'll want proper assets.

Recommended tool: [`@capacitor/assets`](https://github.com/ionic-team/capacitor-assets) — generates every required icon size + splash variants from a single source.

```bash
npm install --save-dev @capacitor/assets
# put a 1024x1024 icon at assets/icon.png and a 2732x2732 splash at assets/splash.png
npx capacitor-assets generate
npx cap sync
```

The current SVG icons (`icon-512.svg`, `favicon.svg`) show "Fund / Huntr" stacked on a green gradient — you can render them to PNG at 1024px to use as the master.

## Plugins in use

Defined in `package.json`. All are official `@capacitor/*` plugins:

| Plugin | What it does |
| --- | --- |
| `@capacitor/core` | Bridge between web app and native shell |
| `@capacitor/ios` | iOS native runtime |
| `@capacitor/android` | Android native runtime |
| `@capacitor/app` | Lifecycle + hardware back button |
| `@capacitor/status-bar` | Color the status bar to match the brand |
| `@capacitor/splash-screen` | Native splash screen on launch |
| `@capacitor/haptics` | Vibration on tap/swipe |

To add a plugin (e.g. push notifications):

```bash
npm install @capacitor/push-notifications
npx cap sync
```

Then import it at the top of `mobile.js`.

## Shipping to the App Store

1. In Xcode → Project → Signing & Capabilities → set your team
2. Bump the version: Xcode → General → Identity → Version + Build
3. Product → Archive
4. When the Organizer opens: Distribute App → App Store Connect → Upload
5. Submit for review in App Store Connect

Pre-launch checklist:

- App icon at 1024×1024 (no transparency, no rounded corners — iOS adds them)
- Screenshots: 6.7", 6.5", 5.5" iPhone sizes minimum
- Privacy policy URL (FundHuntr collects email + profile data)
- App Store description, keywords, support URL

---

## Shipping to Google Play

1. In Android Studio → Build → Generate Signed Bundle / APK
2. Pick **Android App Bundle**
3. Create a new keystore (back it up — losing it means losing the app)
4. Build `release` variant → produces an `.aab` file
5. Upload to Google Play Console → Production track

Pre-launch checklist:

- Adaptive icon (foreground PNG 432×432 + background color)
- Feature graphic 1024×500
- Screenshots in at least one device class
- Privacy policy URL
- Data safety form (declare you collect email + profile info)

## Updating after web changes

The most common workflow:

```bash
# 1. Edit any web file
vim discover.html

# 2. Sync into both native projects
npx cap sync

# 3. Run again in Xcode or Android Studio
```

`npx cap sync` is fast (≈1 second) and idempotent — run it whenever the web app changes.

For ship builds, bump the version in `package.json` AND in the native projects (`Info.plist` for iOS, `build.gradle` for Android), then archive/release.

## Live-reload during development

For faster iteration without `npx cap sync` between every change, point Capacitor at your local dev server:

```bash
# Terminal 1
npm start                                # serves on :8080

# Terminal 2 — get your machine's local IP, then:
npx cap run ios --livereload --external --port 8080
# or
npx cap run android --livereload --external --port 8080
```

The phone now loads from your laptop, with hot reload on every save. Remember to switch back to a baked build before shipping.

## Troubleshooting

- **`Pod install` fails on iOS** → `cd ios/App && pod install --repo-update`
- **Android can't find SDK** → set `ANDROID_HOME` in your shell, or in `android/local.properties`: `sdk.dir=/path/to/Android/sdk`
- **WebView shows blank screen** → check Chrome devtools for Android (`chrome://inspect`) or Safari devtools for iOS — usually a JS error in `app.js` or a missing file in `npx cap sync`
- **Status bar color doesn't change** → make sure `mobile.js` is loaded *after* `app.js` in every HTML file
